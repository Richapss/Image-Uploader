API Documentation

Dokumentasi lengkap untuk mengintegrasikan Image Uploader langsung dari aplikasi atau script kamu menggunakan HTTP Request standar. Tidak perlu membuka website.

Base URL http://localhost:3001
🔓 Autentikasi: Seluruh endpoint upload gambar dan disk status bersifat publik dan tidak memerlukan API Key. Endpoint donasi membutuhkan konfigurasi Payinaja di sisi server (API Key disimpan di file .env dan tidak perlu disertakan oleh pengguna).
☁️ Upload Gambar
POST /upload Upload satu gambar

Mengunggah file gambar ke server. Server akan mengembalikan URL unik berbasis UUID yang dapat langsung dibagikan. Gambar akan otomatis terhapus jika tidak diakses dalam 14 hari.

Request Body (multipart/form-data)
FieldTipeKeterangan
image File File gambar yang akan diunggah (jpg, png, gif, webp, dll)  Wajib
Contoh Request
bash 
curl -X POST http://localhost:3001/upload \
  -F "image=@/path/to/gambar.jpg"
javascript 
const formData = new FormData();
formData.append('image', fileInput.files[0]);

const res = await fetch('/upload', {
    method: 'POST',
    body: formData
});

const data = await res.json();
console.log(data.url);
Response 200 OK
200 OK
json 
{
  "message": "Upload berhasil",
  "id": "a1b2c3d4-...-ef12.jpg",
  "url": "http://localhost:3001/i/a1b2c3d4-...-ef12.jpg"
}
GET /i/:id Ambil / tampilkan gambar

Menampilkan gambar berdasarkan ID unik. Setiap kali endpoint ini dipanggil, timer kedaluwarsa gambar akan direset otomatis +14 hari dari detik akses terakhir.

Path Parameter
ParameterTipeKeterangan
id String ID unik gambar (UUID + ekstensi), misal: a1b2c3d4-ef12.png  Wajib
Contoh Request
bash 
curl http://localhost:3001/i/a1b2c3d4-...-ef12.jpg
Response
200 OK

Mengembalikan file gambar secara langsung (binary stream) yang siap ditampilkan di browser.

💾 Status Server
GET /api/disk-status Cek kapasitas disk server

Mengembalikan data penggunaan disk server aktif secara real-time dalam satuan persen dan Gigabyte.

Contoh Request
bash 
curl http://localhost:3001/api/disk-status
Response 200 OK
200 OK
json 
{
  "usedPercent": 42,
  "freePercent": 58,
  "usedGB": "85.2",
  "freeGB": "114.8",
  "totalGB": "200.0"
}
❤️ Sistem Donasi
POST /api/donate/create Buat QRIS Donasi

Membuat kode QRIS pembayaran donasi melalui Payinaja. QRIS berlaku selama 15 menit. Gunakan trx_id dari response ini untuk polling status pembayaran.

Request Body (application/json)
ParameterTipeKeterangan
amount Integer Nominal donasi dalam Rupiah. Minimal Rp 100  Wajib
name String Nama donatur untuk pencatatan  Opsional
Contoh Request
bash 
curl -X POST http://localhost:3001/api/donate/create \
  -H "Content-Type: application/json" \
  -d '{"amount": 5000, "name": "Budi"}'
Response 200 OK
200 OK
json 
{
  "success": true,
  "trx_id": "TRX-1774870130449-6ND",
  "qris_image_url": "https://quickchart.io/qr?text=...&size=300",
  "amount_requested": 5000,
  "total_amount": 5035,
  "fee": 35
}
GET /api/donate/status/:trx_id Cek status transaksi

Mengecek status pembayaran donasi berdasarkan ID transaksi. Gunakan polling setiap 3 detik setelah QRIS ditampilkan untuk mendeteksi pembayaran berhasil secara otomatis.

Path Parameter
ParameterTipeKeterangan
trx_id String ID transaksi dari response /api/donate/create  Wajib
Contoh Request
bash 
curl http://localhost:3001/api/donate/status/TRX-1774870130449-6ND
Response 200 OK
200 OK
json 
{
  "success": true,
  "status": "success",
  "trx_id": "TRX-1774870130449-6ND",
  "amount": 5000,
  "total_amount": 5035,
  "payment_method": "QRIS",
  "created_at": "2026-04-06T00:00:00+07:00"
}
Nilai Field Status
NilaiArti
pendingMenunggu pembayaran atau belum terkonfirmasi
success✅ Pembayaran berhasil diterima
failed❌ Pembayaran gagal atau QRIS kadaluarsa
GET /api/donate/transactions Riwayat donasi sukses

Mengembalikan daftar riwayat transaksi donasi yang telah sukses terbayar, difilter dari akun Payinaja server.

Query Parameter
ParameterTipeDefaultKeterangan
limit Integer 10 Jumlah data yang ingin diambil  Opsional
Contoh Request
bash 
curl "http://localhost:3001/api/donate/transactions?limit=5"
Response 200 OK
200 OK
json 
{
  "success": true,
